跨时钟域与异步FIFO

DIC-010：跨时钟域与异步FIFO

Mar 25, 2026
阅读时长: 22 分钟

本文主要讨论数字IC设计中的跨时钟域（Clock Domain Crossing, CDC）问题与异步FIFO（Asynchronous FIFO）电路设计。跨时钟域问题其通常指在数字系统存在多个独立时钟域时，应当如何安全、稳定且高效地实现不同时钟域间的信号传输，而异步FIFO则是解决跨时钟域问题的一种非常成熟且实用的电路模块。

跨时钟域

跨时钟域的风险及其分类

当进行跨时钟域信号传输时，可能出现的风险分为两类，但引发这两类风险的原因是一致的，即：当使用目的时钟2驱动触发器采样异步信号时，异步信号与采样时钟的 “时序” 3是随机变化的，这种随机性可能引发电路不确定的行为，包括：

  • 亚稳态（Metastable）： 若异步信号变化的时刻与目的时钟采样边沿过于靠近，致使采样行为不满足建立/保持时间约束，则触发器的采样输出可能是一个介于1与0之间的非稳态电平，且在后续多个目的时钟周期内都可能无法收敛至稳态电平，这主要是触发器的结构导致的，因为触发器内部通常包含组合逻辑门构成的反馈环路，其除了输出1或0的稳定状态外，一般会存在一个输出中间电平的亚稳态，这里“亚”的含义是：当外部无扰动时，触发器可以稳定保持该状态，但若外部发生任何电平波动，触发器会通过反馈环路触发正反馈，在有限时间内使内部状态收敛至某一稳态4。

  • 采样随机性（Randomness）： 即使触发器的采样没有诱发亚稳态，电路仍会存在不确定行为，包括两点：

    • 在采样行为不满足建立/保持时间约束时，我们无从确定触发器的输出是1还是0；

    • 即使采样行为满足时序约束，采样得到的序列也一定会随异步信号与目的时钟的相对相位变化而变化（比如对一个脉宽仅有1个目的时钟周期的异步信号而言，目的时钟是有概率无法采样到该信号有效值的）。

    换言之，采样异步信号存在天然的不确定性，采样随机性会引申出诸如数据漏采、同步失序等一系列问题，因此这也是跨时钟域信号传输时主要需要解决的问题，在设计数字系统时，我们不希望系统内部行为存在任何不可预测的特征。

解决亚稳态的方法

最常见的方法是使用多个触发器级联构成的 同步器（Synchronizer） 实现信号采样。但需要注意，即使使用同步器，也仍然存在非常小的可能诱发亚稳态。工程人员通常使用 平均无故障工作时间（Mean time between failures, MTBF） 这一因子评估单个触发器诱发亚稳态的可能性，对其的一个直观认识是：其通常反比于采样时钟频率$f_\text{clk}$与异步信号翻转频率$f_\text{data}$的乘积，即：

$$ f\propto \frac{1}{f_\text{clk}\times f_\text{data}} $$

因此，当多个触发器级联构成同步器时，同步器的MTBF会随级联的触发器个数增加而近似呈指数衰减5。

另外，有关同步器的一个重要认识是：同步器的输出信号一定是被采样异步信号的历史状态，这很容易理解，因为同步器在功能上就是将输入信号打了若干拍。这一认识在异步FIFO设计中非常重要，我们将在后文进行说明。

解决采样随机性的方法

同步器无法消除采样随机性，因其本质上仍是在通过目的时钟采样一个在时序上完全随机的异步信号。因此，采样随机性的消除必须要引入额外的电路设计。而从之前的分析也可看出，这一额外电路的主要功能即是对异步信号施加额外的时序约束，因为只有输入异步信号的行为受到约束，我们才有可能实现确定性的采样。

当然，实现这一效果的方法是多样的，对应适用的场景也是不同的，我们可以分别进行简单讨论：

单路信号跨时钟域

单路信号跨时钟域传输的主要风险是采样随机性引发的数据漏采，即我们无法保证目的时钟一定能采样到异步信号的有效值。

当只有单路信号进行跨时钟域传输时，我们可以通过展宽信号的方式确保目的时钟至少能够正确采样1次信号，从而确保信号的正确传输，该方法需要注意以下事项：

  • 确保目的时钟正确采样的前提是展宽后的信号脉宽需满足所谓的 “三边沿”要求 6，即信号有效的时间范围内至少要包含3个目的时钟的边沿，由于我们通常仅会统一选取时钟上升沿/下降沿中的一者进行采样，因此“三边沿”要求可以保证目的时钟至少能正确采样信号1次。同时，为保证信号有效的时间范围完全包含目的时钟的3个边沿，我们通常要求信号脉宽不小于1.5倍的目的时钟周期。
  • 信号脉宽不小于1.5倍的目的时钟周期是一个粗略约定，如果我们希望确保目的时钟域采样到正确的信号值，可以采用闭环控制的方式：目的时钟域将同步后的信号再次发送至发送时钟域，由发送时钟域同步并对比该信号与初始发送信号是否一致，在此过程中，初始发送信号始终保持不变，直到对比完成后才可变化。这一过程类似于两时钟域进行一次简单的握手，安全性高，但实时性不高。
  • 即使对信号进行展宽后，我们仍无法保证目的时钟域接收到信号的有效周期数是固定值，因为展宽信号无法消除异步信号与目的时钟相对相位的随机性，因此我们无法确定展宽后信号被目的时钟采样的次数。换言之，在单路信号跨时钟域传输时，我们只能传输那些在目的时钟域对有效周期数没有严格要求的信号，诸如 “通过跨时钟域传输某信号，使其在目的时钟域被同步为单周期脉冲” 的需求是不合理的。如果在某些设计中，确实需要满足上述需求，我们可以考虑跨时钟域传递一个多周期的使能信号，随后在目的时钟域利用同步后的使能信号触发脉冲生成。这一方法与后文中逻辑后移的思路是相同的。

多路信号跨时钟域

多路信号跨时钟域传输的主要风险是各路信号因采样随机性引发的数据漏采，以及各路信号间因采样随机性引发的同步失序。其中，同步失序产生的原因可归纳为：各路信号在跨时钟域传输时均存在独立的采样随机性，那么多路信号同时传输时，我们显然不能认为各路信号随机性的“偏向”是一致的，因此各路信号在采样后的时序可能发生变化。

消除多路信号跨时钟域的风险的原则是：将其转化为单路信号跨时钟域的场景，并运用单路信号跨时钟域的电路设计消除风险。

方法1：逻辑后移

逻辑后移其实是笔者自己归纳的一种方法，对应着Cliff Cummings文档中的多个电路解决方案，但笔者认为它们在设计思路上是一致的。

逻辑后移的定义如下：如果目的时钟域对跨时钟域信号的时序有严格要求，同时在发送时钟域，存在某一子模块用于产生跨时钟域的所有信号，且该子模块仅由发送时钟域的单一信号驱动7，则可以将该子模块整体迁移至目的时钟域，并将驱动该子模块的信号作为单路跨时钟域信号进行传输。

比如，若要传输多路完全相同的信号，那么完全可以仅对其中一路信号进行跨时钟域传输，随后在目的时钟域将其复制为多份。

再比如，若要求传输的信号A与信号B正好差一拍，那么可以仅对信号A进行跨时钟域传输，随后在目前时钟域打一拍生成信号B。

逻辑后移从本质上看是从空间上将多路信号跨时钟域转化为单路信号跨时钟域，即通过改变模块所在时钟域的方式进行转化；对称地，我们也可以从时间上转化，即不改变模块所在时钟域，但通过引入额外逻辑，使得在各个采样时间点观察跨时钟域传输的多路信号，应当有且仅有一路信号可能发生变化。此时，各采样时间点只会有一路信号存在采样随机性，这便转化为了单路信号跨时钟域的场景，进而避免了同步失序，后续的各种解决方案都或多或少使用了这种思路。

这里需要明确一点，什么叫做“在某个采样时间点观察多路跨时钟域信号，有且仅有一路信号可能变化”？更为具体的定义是：在某个采样时间点不满足建立/保持时间约束的时间范围内，应当有且仅有一路信号可能变化，因为这样才可保证只会有一路信号存在采样随机性。但不难发现，这要求发送时钟周期$T_\text{tran}$，建立时间$T_\text{setup}$与保持时间$T_\text{hold}$满足如下约束：

$$ T_\text{tran}>T_\text{setup}+T_\text{hold} $$

在较为先进的工艺下，建立/保持时间通常在10ps这个量级，而系统时钟周期通常在1ns这个量级，因此上述不等式在大多数情况下是平凡成立的。换言之，我们只需要保证在每个发送时钟周期内，均只有且仅有一路信号可能发生变化即可。当然，如果发送时钟周期非常小，以至于其小于建立/保持时间之和，那么下述对多路跨时钟域信号的处理方法就都失效了。

方法2：独热码

这种方法的思路是：将$N$路信号视为一个$N$位的二进制数，随后将其编码为$2^N$位的独热码（One-Hot Code, $N$位二进制数最多存在$2^N$个取值）进行跨时钟域传输，最后在目的时钟域解码还原为$N$路信号。

独热码在切换时，每次都有且仅有一位由0变1，有且仅有1位由1变0，此时仍有两路跨时钟域信号发生变化，似乎与前文提及的有且仅有一路信号可能发生变化的要求矛盾。但我们不难发现，对于独热码而言，只有由0变1的那一路信号决定了独热码切换后的取值，因此，我们在目的时钟域设计解码模块时，可以忽略从1变0的那一路信号，即：

  • 将独热码编码的多路数据分别通过同步器同步至目的时钟域，输入解码模块；
  • 解码模块通过有限状态机（FSM）实现，状态即为当前采样时间点解码的二进制数，随后，在下一采样时间点，如果检测到输入的二进制码的某位由0变1（至多有一位），便根据该路所在位更新至下一状态实现解码。

这里需要指出，采用独热码编/解码的方法是为了解决同步失序，但各路信号在跨时钟域传输时，不能忘记考虑数据漏采，即各路信号均需展宽，满足 “三边沿”要求 。

方法3：DMUX同步器

独热码方法的缺点是，对独热码编码后的各路信号，我们都需要插入触发器构成的同步器进行同步，当跨时钟域的信号位宽较大时，这会引发相当大的资源开销。

为此，一个可行的方法是使用DMUX同步器，这种方法的思路是：

  • 跨时钟域传输的多路信号不使用同步器，通过发送时钟域直接传输至目的时钟域；
  • 同时，额外引入一路控制信号，使用同步器进行跨时钟域传输，并在目的时钟域转化为脉冲信号触发对多路信号的采样；
  • 在目的时钟域对多路信号完成采样前，多路信号不能发生跳变，这需要上层引入额外的控制逻辑环路实现。

上图展示了Cliff Cummings文档中给出的一类典型的DMUX同步器设计（在文档中，这个电路不叫DMUX同步器，而被称为MCP formulation toggle-pulse generation，但在功能上等价于DMUX同步器），Data In为需要跨时钟域传输的多路数据信号，Next Data Pulse则为额外引入的控制信号，该电路的工作原理如下：

  • Next Data Pulse信号在每次数据有效时产生一个单周期脉冲，该单周期脉冲一方面会在发送时钟域将有效数据载入寄存器中，使得该寄存器输出在下次有效数据产生前保持不变；另一方面会触发控制信号的输出逻辑（XOR门与D触发器构成的反馈结构，功能上相当于一个脉冲触发的分频器），使其发生一次翻转，将脉冲式的有效信号转化为常高/低式的信号输出；
  • 输出的控制信号经目的时钟域同步器同步后，会再次被转化为单周期脉冲（图中同步器与脉冲生成模块被封装为Load Data Pulse Gen模块，脉冲生成模块也可通过XOR门与D触发器实现），用于触发目的时钟域的寄存器采样数据，完成多路信号的跨时钟域传输。

DMUX同步器的缺点是，为保证在目的时钟域对多路信号完成采样前信号不发生跳变，需要额外的控制逻辑，这会降低信号传输的实时性；同时，闭环的控制逻辑需要发送/目的时钟域的协同设计，这不利于两时钟域的逻辑解耦。

方法4：异步FIFO

最成熟、最稳定的方案，我们将在后文进行说明。

跨时钟域的时序约束

跨时钟域的时序约束需要注意以下三点：

  • 为了简化时序约束和分析，在划分电路模块时，要求每个模块只允许存在一个时钟；同时，创建独立的同步器模块，用于处理某两个时钟域间的信号传递。此时，除了同步器模块外，其余功能模块为纯粹的同步设计，可独立进行简单的时序分析。
  • 对于跨时钟域传输的信号，静态时序分析（STA）的结果是不准确且无意义的。因为目的时钟与异步信号的相对相位是变化的，理论上需要检查无限多种相位关系，故需要使用set_false_path令综合工具禁用这些路径的时序检查8。
  • 对于同步器模块本身，需要对第一级触发器到第二级触发器间的路径进行最小延迟（Min-Time）分析，确保其满足保持时间约束。

异步FIFO

异步FIFO的相关问题

空/满信号的判定及所在时钟域

异步FIFO的空信号（empty）在读时钟域，通过对比读指针与跨时钟域传输的写指针生成；异步FIFO的满信号（full）在写时钟域，通过对比写指针与跨时钟域传输的读指针生成。

空信号处于读时钟域有两方面优点：

  • 只有对FIFO的读操作才会使用空信号，故无需额外的跨时钟域操作同步该信号；
  • 当读指针追上写指针时，FIFO为空，但由于在读时钟域，跨时钟域传输的写指针始终保存的是写指针的历史状态（前文提到过！），所以无论读/写时钟的频率关系如何，读时钟域判定空信号有效始终是异步FIFO为空的必要不充分条件（即当异步FIFO为空时，空信号一定有效，但空信号有效时异步FIFO不一定为空）。这可以确保不会出现异步FIFO为空，但空信号无效的风险情形，该情形一旦出现，上层逻辑可能从异步FIFO中读出非法值。

对称地，满信号处于写时钟域有两方面优点：

  • 只有对FIFO的写操作才会使用满信号，故无需额外的跨时钟域操作同步该信号；
  • 当写指针追上读指针时，FIFO为满，但由于在写时钟域，跨时钟域传输的读指针始终保存的是读指针的历史状态，所以无论读/写时钟的频率关系如何，写时钟域判定满信号有效始终是异步FIFO为满的必要不充分条件（即当异步FIFO为满时，满信号一定有效，但满信号有效时异步FIFO不一定为满）。这可以确保不会出现异步FIFO为满，但满信号无效的风险情形，该情形一旦出现，上层逻辑可能对异步FIFO执行无效的写入操作。

读/写指针的编码、同步、对比与指针-地址编码转换

异步FIFO的读/写指针有两种功能：

  • 用于判定异步FIFO的空/满状态。
  • 用于作为地址寻址读/写异步FIFO内部的存储器。

当读/写指针用于判定异步FIFO的空/满状态时，对指针有两点要求：

  • 由于判定空/满状态时，写/读指针分别需要进行跨时钟域同步，为避免同步失序，其通常使用格雷码（Gray Code）进行编码，由于读/写指针一定是以1为单位循环递增的，而相邻格雷码的码距9为1，因此这同样是一种从时间上将多路信号转化为单路信号跨时钟域的方法。
  • 由于判定空/满状态需要区分读/写指针的相互追及关系，因此对深度为$2^N$的FIFO，我们需要分别使用$N+1$位信号编码读/写指针，其中MSB为标志位，用于标识指针循环递增周期数的奇偶性，当读/写指针标志位相同时，代表读指针追上写指针，FIFO为空；当读/写标志位不同时，代表写指针追上读指针，FIFO为满。

综上，对深度为$2^N$的FIFO，我们需要分别使用**$N+1$位格雷码**编码读/写指针，但是，对格雷码而言，我们不能按二进制码的思路，仅仅通过 “相等” 判断读/写指针重合，而需结合格雷码具体的性质进行判断：

  • $N+1$位格雷码前$2^N$个取值的MSB均为0，后$2^N$个取值的MSB均为1，这与二进制编码相同，因此在通过格雷码区分读/写指针的相互追及关系时，可以直接对MSB使用 “相等” / “不等” 进行判定。
  • $N+1$位格雷码前$2^N$个取值的低$N$位，与后$2^N$个取值的低$N$位存在如下关系：低$N-1$位对应相等，第$N$位对应相反。

因此，我们判定空/满状态的逻辑应当为：

1
2

assign empty = (read_ptr == write_ptr_sync);
assign full  = ({~write_ptr[N-:2], write_ptr[N-2:0]} == read_ptr_sync);

以下，我们还需要额外考虑读/写指针的同步问题，异步FIFO中，我们只是简单使用同步器对读/写指针进行同步，但是没有对其进行信号展宽，这可能导致以下两个后果：

  • 读/写指针的某些取值可能被漏采；
  • 当读/写指针的某些取值被漏采时，即使读/写指针使用格雷码编码，也不能保证当前采样时间点的指针与上一采样时间点的指针对应编码码距为1。

但上述后果并不会影响异步FIFO的功能正确性，原因如下：

  • 即使读/写指针被漏采，被同步至读/写时钟域的写/读指针仍是真实指针的历史状态，因此不会产生异步FIFO误判空/满的风险情形。
  • 读/写指针被漏采时，虽然当前采样时间点的指针与上一采样时间点的指针对应编码码距不一定为1，但是在当前采样时间点观察指针，仍然只可能有一位发生变化（可以理解为虽然对比前一采样时间点与当前采样时间点，指针有多位发生变化，但是指针仍是一位一位发生变化的），而对变化的这一位，无论采样得到其变化前的值还是变化后的值，整体取值都对应着有效的格雷码。

module rptr_empty #(
    parameter ADDRSIZE = 4
) (
    input  wire                rclk,
    input  wire                rrst_n,

    input  wire                rinc,
    input  wire [ADDRSIZE:0]   rq2_wptr,

    output reg                 rempty,
    output wire [ADDRSIZE-1:0] raddr,
    output reg  [ADDRSIZE:0]   rptr
);

/* Binary address counter*/
reg [ADDRSIZE:0] rbin; // use ADDRSIZE counter

always @(posedge rclk or negedge rrst_n) begin
    if (!rrst_n) begin
        rbin <= 'b0;
    end else begin
        rbin <= rbin + ((!rempty) && (rinc));
    end
end

assign raddr = rbin[ADDRSIZE-1:0];

/* Gray-coded read pointer -use rptr */
wire [ADDRSIZE:0] next_rptr;

assign next_rptr = (rbin >> 1) ^ (rbin);

always @(posedge rclk or negedge rrst_n) begin
    if (!rrst_n) begin
        rptr <= 'b0;
    end else begin
        rptr <= next_rptr;
    end
end

/* Empty detection */
always @(posedge rclk or negedge rrst_n) begin
    if (!rrst_n) begin
        rempty <= 1'b0;
    end else begin
        rempty <= (next_rptr == rq2_wptr);
    end
end

// FIFO property config
parameter DATASIZE = 8;
parameter ADDRSIZE = 4;
parameter FIFO_DEPTH = 1 << ADDRSIZE;

// R&W clock period config
parameter WCLK_PERIOD = 10;
parameter RCLK_PERIOD = 13;

/* Variables setting */
// Test summation variables 
integer write_count = 0;
integer read_count  = 0;
integer error_count = 0;

// FIFO port variables
reg                 wclk, rclk, wrst_n, rrst_n, winc, rinc;
reg  [DATASIZE-1:0] wdata;

wire                wfull, rempty;
wire [DATASIZE-1:0] rdata;

// test sequence
reg  [DATASIZE-1:0] data_queue [FIFO_DEPTH*2:0]; // REF
integer             queue_head = 0;
integer             queue_tail = 0;

// temp value
reg  [DATASIZE-1:0] read_val;
reg  [DATASIZE-1:0] expected_val;

/* Module instantiate */
async_fifo #(
	.DATASIZE  (DATASIZE  ),
	.ADDRSIZE  (ADDRSIZE  )
) u_async_fifo (
	.wclk    (wclk    ),
	.rclk    (rclk    ),
	.wrst_n  (wrst_n  ),
	.rrst_n  (rrst_n  ),
	.winc    (winc    ),
	.wdata   (wdata   ),
	.wfull   (wfull   ),
	.rinc    (rinc    ),
	.rdata   (rdata   ),
	.rempty  (rempty  )
);

/* Clock generation */
initial begin
    wclk = 0;
    forever #(WCLK_PERIOD / 2) wclk = ~wclk;
end

initial begin
    rclk = 0;
    forever #(RCLK_PERIOD / 2) rclk = ~rclk;
end

/* Test task definition */
task automatic write_data;
    input [DATASIZE-1:0] data;
    begin
        @(posedge wclk); winc = 1'b1;
        wdata = data;
        @(posedge wclk); winc = 1'b0;
        write_count = write_count + 1;
        $display("[%0t] WRITE: data = 0x%02h, count = %0d", $time, data, write_count);
    end
endtask

task automatic read_data;
    output [DATASIZE-1:0] data;
    begin
        data = rdata; // this FIFO is Frist-Word Fall-Through FIFO
        @(posedge rclk); rinc = 1'b1;
        @(posedge rclk); rinc = 1'b0;
        read_count = read_count + 1;
        $display("[%0t] READ:  data = 0x%02h, count = %0d", $time, data, read_count);
    end
endtask

task automatic push_queue;
    input [DATASIZE-1:0] data;
    begin
        data_queue[queue_tail] = data;
        queue_tail = queue_tail + 1;
    end
endtask

task automatic pop_queue;
    output [DATASIZE-1:0] data;
    input                 check_empty;
    begin
        if (check_empty && (queue_head >= queue_tail)) begin
            $display("[%0t] ERROR: Queue empty but read attempted!", $time);
            error_count = error_count + 1;
        end else begin
            data = data_queue[queue_head];
            queue_head = queue_head + 1;
        end
    end
endtask

task automatic verify_data;
    input  [DATASIZE-1:0] expected;
    input  [DATASIZE-1:0] actual;
    begin
        if (expected != actual) begin
            $display("[%0t] ERROR: Data mismatch! Expected = 0x%02h, Got = 0x%02h", $time, expected, actual);
            error_count = error_count + 1;
        end else begin
            $display("[%0t] PASS: Data verified 0x%02h", $time, actual);
        end
    end
endtask

    if (rempty !== 1'b0) begin
        $display("ERROR: FIFO should not be empty after writing!");
        error_count = error_count + 1;
    end else begin
        $display("PASS: FIFO is not empty after writing");
    end

    // Read & verify 5 data
    for (integer i = 0; i < 5; i = i + 1) begin
        pop_queue(expected_val, 0);
        read_data(read_val);
        verify_data(expected_val, read_val);
        #(RCLK_PERIOD * 2);
    end

    // Check empty signal
    #(RCLK_PERIOD * 3);
    if (rempty != 1'b1) begin
        $display("ERROR: FIFO should be empty after reading all data!");
        error_count = error_count + 1;
    end else begin
        $display("PASS: FIFO is empty after reading all data");
    end

    $display("\n[TEST 4] Full Flag Test");
    $display("------------------------------");

    // Write until full
    for (integer i = 0; i < FIFO_DEPTH; i = i + 1) begin
        write_data(8'hA0 + i);
        push_queue(8'hA0 + i);
        #(WCLK_PERIOD * 2);
    end

    // Check full signal
    #(WCLK_PERIOD * 3);
    if (wfull != 1'b1) begin
        $display("ERROR: wfull should be 1 when FIFO is full!");
        error_count = error_count + 1;
    end else begin
        $display("PASS: wfull = 1 when FIFO is full");
    end

    // Write when FIFO is full (should be ignored)
    write_data(8'hFF);
    #(WCLK_PERIOD * 3);

    $display("\n[TEST 5] Concurrent Read/Write Test");
    $display("------------------------------");

    // Read half data
    for (integer i = 0; i < FIFO_DEPTH / 2; i = i + 1) begin
        pop_queue(expected_val, 0);
        read_data(read_val);
        verify_data(expected_val, read_val);
        #(RCLK_PERIOD * 2);
    end

    // Check full signal
    #(WCLK_PERIOD * 3);
    if (wfull != 1'b0) begin
        $display("ERROR: wfull should be 0 after reading from full FIFO!");
        error_count = error_count + 1;
    end else begin
        $display("PASS: wfull = 0 after reading from full FIFO");
    end

    $display("\n[TEST 6] Reset During Operation Test");
    $display("------------------------------");

    // Reset working FIFO
    wrst_n = 1'b0;
    rrst_n = 1'b0;
    #50;
    wrst_n = 1'b1;
    rrst_n = 1'b1;
    #50;

    // Check reset status
    if (rempty != 1'b1) begin
        $display("ERROR: rempty should be 1 after reset during operation!");
        error_count = error_count + 1;
    end else begin
        $display("PASS: rempty = 1 after reset during operation");
    end

    #100;
    $display("\n=============================================");
    $display("                Test Summary                 ");
    $display("=============================================");
    $display("Total Write Operations: %0d", write_count);
    $display("Total Read Operations:  %0d", read_count);
    $display("Total Errors:           %0d", error_count);
        
    if (error_count == 0) begin
        $display("=============================================");
        $display("               ALL TESTS PASSED!             ");
        $display("=============================================");
    end else begin
        $display("=============================================");
        $display("   TESTS FAILED! Please check errors above   ");
        $display("=============================================");
    end
    
    $finish;
end

/* Waveform generation */

    $dumpfile("tb_async_fifo.fsdb");
    $dumpvars(0, tb_async_fifo);

    $display("ERROR: Simulation timeout!");
    $finish;
